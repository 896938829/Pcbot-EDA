## 1. 背景

## 2. 检查项（草案）

### 2.1 PCB 视口基础（P6）

- [ ] F5 → 载 demo `led_blink.pcbproj` → 切到 PCB 视口 → 看到板形外框 + 5 个元件 footprint 完整渲染（pads + outline）
- [ ] 层切换 toggle：top / bottom / silkscreen / courtyard 各层独立显示 / 隐藏正确
- [ ] 滚轮 zoom → StatusBar zoom% 同步（同 M1.2.1 P4 模式）
- [ ] 中键 pan → 元件随之移动，超出视口被 `clip_contents` 裁掉
- [ ] 网格按 zoom level 自适应密度

### 2.2 放置编辑（P6）

- [ ] 左键点击 footprint 中心 → 橙色选中 + PropertiesPanel 显示属性
- [ ] 拖拽选中 footprint → 落点调用 `pcb.move_footprint`，`.pcbot/last-run.json` 记录
- [ ] 旋转 hotkey（R）→ 90° 步进，调用 `pcb.rotate_footprint`
- [ ] Delete → 调用 `pcb.remove_footprint`
- [ ] Ctrl+Z / Ctrl+Y → Undo/Redo 完整还原（含 net 关联）

### 2.3 手动走线（P6）

- [ ] 选走线工具 → 点击焊盘 A（VCC net）→ 鼠标移动看直角折线预览
- [ ] 点击焊盘 B（同 VCC net）→ 调用 `pcb.add_trace`，走线落盘可见
- [ ] 点击焊盘 C（不同 net）→ 命令拒绝 + StatusBar 提示 `PCB_NET_MISMATCH`
- [ ] 走线中段右键 → 添加 via，调用 `pcb.add_via`
- [ ] 选中走线 + Delete → `pcb.remove_trace`，含 inverse 入 Undo

### 2.4 DRC 高亮（P5+P6）

- [ ] CliConsole 跑 `drc.basic` → 结果回显
- [ ] 故意挪一个元件造成 courtyard 冲突 → DRC 视口红框 + `diagnostics.jsonl` 追加
- [ ] 修复后再跑 → 红框消失 + jsonl 追加 `resolved_by_commit` 行
- [ ] 点击红框 → 视口聚焦 + PropertiesPanel 显示违例详情

### 2.5 元件库性能（P1）

- [ ] 切换到大型库（1500+ 条种子）→ `library.search` 响应 < 200ms（StatusBar 显示耗时）
- [ ] 库面板搜索框输入 → 增量过滤无明显卡顿
- [ ] 修改一个 `*.json` 元件文件 → mtime 增量扫描生效，重启不重扫全量

### 2.6 AI 走线工作流（P7）

- [ ] 按 `docs/specs/2026-04-23-ai-routing-workflow.md` 准备 prompt
- [ ] 让 Claude / Codex 通过 CLI 完成 led_blink 5 元件全部信号网络走线
- [ ] AI 输出最终 `drc.basic` 报 0 违例
- [ ] 全程无需人工干预（除初次 prompt 与失败重试）

### 2.7 跨视口协同（P6）

- [ ] 原理图视口删除一个元件 → 切到 PCB 视口，对应 footprint 标黄"未关联"
- [ ] PCB 视口走线后 → CliConsole 跑 `schematic.list_nets` 看 connected 状态同步
