## 1. 范围声明

**范围内**：M1 实施过程中为了赶通而延后的技术债。每一项都能在 M2 前独立完成、
独立验收，不引入新域（PCB / DRC）。

**范围外**：

- 任何 M2 属性的功能（PCB 数据模型、DRC 规则引擎、3D 预览）
- 未在 M1 做完的"新特性"（如 UI 编辑能力增强，留给 M2）
- 跨 EDA 互操作（M4+）

## 2. 验收标准

M1.1 完成需同时满足：

1. **测试基线双轨打通**：装好 GUT 的环境下 `godot --headless -s addons/gut/gut_cmdln.gd -gdir=res://tests/ -gexit` 全绿；
   未装 GUT 仍可跑 `tests/lightweight_runner.gd` 绿。
2. **元件库 SQLite 达标**：`LibraryIndex` 读写走 SQLite；`library.search` 在 10k 条测试数据上 < 100 ms
   （对齐架构设计 §10 红线）；`LibraryIndex.load_from_root` 仍可从 JSON 源重建。
3. **Symbol 真 SVG**：`symbol.export_svg` 输出真实几何（线段 + 引脚 + 名称），不是占位矩形；
   `SchematicView` 加载该 SVG 替代目前的方块代理。
4. **CLI 作为子进程跑通**：`tests/integration/cli_stdin_test.gd` 用 `OS.execute` 起 `godot --headless -s cli/main.gd`，
   stdin 喂 JSON-RPC，stdout 校验响应，退出码与 PcbotError 一致。
5. **性能基线入库**：`tests/perf/BASELINE.md` 记录当前 1k 元件 / 10k 网络 / 10k 库条目下的 round-trip、check.basic、library.search 耗时；
   作为 M2 CI 回归门禁的初始值。
6. **格式化门禁就绪**：`gdformat --check Runtime cli tests` 全绿；`tools/fmt.sh` 一键跑；
   CLAUDE.md 与代码规范指向此命令。
7. **`Runtime/core/di.gd` 决断**：实现或删除。实现则至少一个模块（建议 `SchematicCommands`）走 DI；
   删除则更新 [架构设计 §2](../architecture/架构设计.md) 并加 ADR。
8. **`annotate` 规则完整**：按 reference 前缀（`R?` / `U?` / `C?` ...）独立分配计数；
   支持预留号段（`start_at` 参数）；用例覆盖混合已命名与未命名的场景。
9. **文档对齐**：本计划、CLAUDE.md、架构设计、代码规范、Skills YAML 一致；偏差已 ADR 记录。

## 3. 工作分解（WBS）

按优先级与依赖排列。规模：`S`（数小时）/ `M`（1–3 天）/ `L`（3 天+）。

### P1 — GUT 接入与迁移（S）

- `addons/gut/` 子模块（`git submodule add https://github.com/bitwes/Gut.git addons/gut`）
- 启用 GUT 插件（`project.godot` `[editor_plugins]`）
- 把现有 `tests/unit/*_test.gd` 加 GUT 兼容入口：
  - 保留 `static func run() -> Array`（轻量 runner 用）
  - 新增 `extends GutTest` + `func test_*()` 桥接，内部调 `run()` 并 asserts
- `tools/run_tests.sh`（检测 GUT 存在即用，否则 fallback 轻量 runner）
- 关 ADR-0003 的 M2 切换条款；改写为"GUT 主路径，轻量 runner 作 CI 之前的 smoke"

### P2 — SQLite 元件库（M）

- 引入 `godot-sqlite` GDExtension（`addons/godot-sqlite/`），Windows/Linux 双平台二进制入 LFS
- `Runtime/io/sqlite.gd` 封装：连接、事务、schema 初始化
- `LibraryIndex` 接口保持：内部 JSON 扫描 → SQLite 查询
  - 首次运行检测 `.pcbotlib.sqlite` 不存在或时间戳早于任一 JSON 源，触发重建
  - 10k 条基准数据生成器 `tests/fixtures/gen_lib.gd`
- 关掉 [ADR-0001](../architecture/ADR-0001-library-index-json-scan.md) 或改状态为 "superseded"
- 性能用例：`tests/perf/library_search_perf_test.gd`

### P3 — Symbol 真 SVG（M）

- `Runtime/io/svg.gd`：扩展"从 ComponentSymbol 生成完整 SVG"逻辑
  - 引脚：圆点 + 引脚编号文字 + 名称
  - 外形：按 `bbox_nm` 画矩形（默认）；metadata 指定 `outline_shape` 时支持 circle / polygon
  - 方向指示：根据 `dir` 画短线伸出
- `symbol.export_svg` 同步扩展
- `Runtime/ui/schematic_view.gd`：加载 SVG
  - `Image.load_svg_from_buffer` → `ImageTexture` → 缓存
  - 布局：SVG 贴在 placement.pos_nm 处，缩放与 `_zoom` 联动
- 失败回退：SVG 缺失时显示当前矩形占位（保持可见）
- 用例：`tests/unit/svg_export_test.gd` 验证一个 NE555 符号 SVG 有正确 pin 数 + viewBox

### P4 — CLI stdin 集成（S）

- `tests/integration/cli_stdin_test.gd`：
  - 用 `OS.execute("godot", ["--headless", "--path", ".", "-s", "cli/main.gd"])` 启子进程
  - 通过临时文件 + `OS.read_string_from_stdin` 机制（或 argv 第二路径）喂 JSON-RPC 请求
  - 断言 stdout 是合法 JSON-RPC 响应；退出码与 `errors[]` 一致
- `cli/main.gd`：审查 stdin 读法在 Windows 下的阻塞/EOF 处理；加 `--input <file>` 备选入口（便于测试）
- `docs/skills/run/last.yaml`：补充 stdin vs argv 两种调用示例

### P5 — 性能基线（S）

- `tests/perf/` 目录建立
- 合成数据：1k 元件 / 5k 网络 / 10k 库条目
- 用例：
  - `sch_roundtrip_perf_test.gd` — 写 → 读 → 写 两次字节相同，且第二次写 < 200 ms
  - `check_basic_perf_test.gd` — check.basic 在 1k 元件下 < 50 ms
  - `library_search_perf_test.gd` — 见 P2
- `tests/perf/BASELINE.md`：记录机器、Godot 版本、基线数值；M2 CI 以此为回归门禁

### P6 — 格式化与工具链（S）

- 本仓库安装 `gdformat` / `gdlint`（或 CI 镜像预装）
- `tools/fmt.sh`：`gdformat Runtime cli tests`（写入）
- `tools/fmt_check.sh`：`gdformat --check Runtime cli tests`
- CLAUDE.md 加 "格式化" 小节，指向这两个脚本
- [代码规范 §12](../conventions/代码规范.md) 更新：推荐 → 强制（M2 起加 CI 门禁，M1.1 只手动）

### P7 — DI 容器决断（S）

选一条线：

- **线 A（实现）**：实现 `Runtime/core/di.gd` 最小 DI（按 `class_name` 注册 / 查找 / 替换）；
  `SchematicCommands` 通过 DI 注入 `JsonStable` / `DiagnosticsLog`，便于测试 mock。
- **线 B（删除）**：去掉 [架构设计 §2](../architecture/架构设计.md) 里的 `di.gd` 条目；加 ADR-0004 说明"M1 规模下 DI 容器收益小于复杂度"。

决策写入本计划最终版 §5。

### P8 — annotate 规则补全（S）

- `SchematicCommands._annotate`：
  - 按 reference 前缀分桶（正则 `([A-Za-z]+)\?$`）
  - 参数 `start_at`：`{"R": 100, "U": 200}` 形式指定起始号
  - 跳过已占用号（避免与已命名元件冲突）
- 测试：
  - 已有 `R1 / R3`，新增 `R? / R?` → 分别得 `R2 / R4`（占用 R1/R3 且 R2 可用则 R2）
  - `start_at` 生效
- Skills YAML `docs/skills/schematic/annotate.yaml` 扩参数说明

### P9 — 验收与文档对齐（S）

- 跑 GUT + 轻量 runner 双路径全绿
- 更新 [实现计划-M1](实现计划-M1.md) 状态为 stable（若尚未）
- 更新 CLAUDE.md：加"运行测试"、"格式化"段
- ADR 复盘：
  - ADR-0001 关闭或迁移
  - ADR-0003 改状态为 "superseded by M1.1 测试双轨"
  - 新增 ADR-0004（若 P7 选线 B）
- 本计划置 stable

## 6. 风险清单

| 风险 | 概率 | 影响 | 缓解 |
| --- | --- | --- | --- |
| godot-sqlite 跨平台二进制维护成本 | 高 | SQLite 推不动 | 保留 JSON 回退路径；若 Linux 上稳定但 Windows 不齐，先只对 Linux 启用 SQLite + Windows 继续 JSON |
| SVG 渲染拖慢 `SchematicView` | 中 | 大型原理图卡顿 | Texture 缓存 + `_zoom` 下降到阈值以下时回退矩形 |
| CLI 子进程在 Windows 路径含空格 | 中 | 测试不稳 | 统一走 `ProjectSettings.globalize_path` + quoted args |
| gdformat 与 Godot 内部风格冲突 | 低 | 代码大面积变动 | 首次跑完锁 baseline，之后增量；不把一次性大 diff 塞在同一 commit |
| DI 决断拖延 | 中 | 架构文档持续漂移 | 本计划强制 P7 在 P9 前拍板 |

## 8. 与 M2 的边界

本计划**不**覆盖以下（交 M2）：

- PCB 数据模型与 `*.pcb.json`
- DRC / ERC 引擎与空间索引

- CI (GitHub Actions) — M1.1 只做本地脚本，CI 正式化留 M2-P7
- UI 编辑能力（M1.1 保持只读）
