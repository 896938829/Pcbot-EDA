## 1. 背景

M1.2 全部 17 phase 已交付并 commit，但人工手验（`UI 手验清单.md`）发现 7 类缺陷，
严重度分布：**2 个 critical（符号完全不可读 + 中心不可选）+ 3 个交互 bug（zoom%/pan 溢出/旋转）+
2 个补丁（bbcode/undo 覆盖）**。全部 bug 在当前代码中可定位到明确行，无需结构性改动。

M1.2.1 是纯修缺陷 + 提升手验通过率，不扩范围；所有新功能（框选、net 选中、wire 交互、旋转 placement
的引脚坐标变换）推到 M2。

## 2. 范围

**范围内**（按严重度）：

- P1 符号 body 自动 bbox（修根因："只有引脚点，没有电阻/电容/电源/地/芯片图像" + "中心无法选中"）
- P2 SchematicView `clip_contents` = true（修 "拖到中间 View 外面时不会被遮挡"）
- P3 hit-test 用实际绘制 Rect（兜底修中心可选）
- P4 滚轮 zoom 走 `set_zoom()` 以触发 `zoom_changed` 信号
- P5 PropertiesPanel rotation OptionButton 诊断 + 修复 + reference 回车焦点守护
- P6 CliConsole bbcode 转义（`[`/`]` → `[lb]`/`[rb]`）
- P7 remove_placement / connect / disconnect_pin 的 undo inverse 补齐

**范围外**（→ M2）：

- 框选（rubber-band selection）
- net / wire 选中高亮
- 旋转 placement 时引脚 pos 的 2D 旋转变换（当前 pin hit-test 仅支持 rot=0）
- `.sym.svg` 独立美化图形（demo 符号靠 JSON 重建，够用）
- StatusBar 工程列显示文案（doc 写 `led_blink` 实际是 project.name `LED Blink`）——改文档不改代码

## 3. 验收标准

M1.2.1 完成需满足：

1. F5 → 载 demo → SchematicView 7 个元件全部可见完整符号（矩形框 + 引脚 + 编号）——不是只有黄点
2. 左键点击任一元件**中心**（非边缘）→ 橙色选中
3. 滚轮缩放 → StatusBar zoom% 同步
4. 中键 pan 把元件移出 View 视口 → 元件被裁掉，不溢出到左 / 右 / 底 dock
5. 属性面板 rotation OptionButton 改值 → placement 在 view 内旋转（即使 pins 位置暂不跟转）
6. 属性面板 reference 回车 → 不打开 rotation 下拉
7. CLI 调试面板错误响应红色 + 正确响应绿色可见
8. Ctrl+Z 回退 Del 删除 / 连线 / 断线操作
9. `UI 手验清单.md` 所有 [X] 改 [✔]

## 4. 工作分解

规模：S（<2h）/ S+（半天）/ M（1 天）。

### P1 — 符号 body 自动 bbox（S+）

- [Runtime/io/svg.gd:102 `render_symbol`](../../Runtime/io/svg.gd#L102)：若 `bbox_nm == [0,0,0,0]`，
  从 `sym.pins[*].pos` 取 min/max 作为推导 bbox，再外扩 2mm pad。
- 保持 `bbox_nm` 显式填写的优先级（向后兼容）。
- 验收：打开 demo → NE555 显示 10mm×10mm 矩形 + 8 引脚；R-10k / C / LED 显示窄矩形。
- 单元：在 `tests/unit/svg_export_test.gd` 加一条 "bbox_nm 全零时从 pins 推导 viewbox"。

### P2 — SchematicView clip_contents（S）

- [Scenes/Main.tscn](../../Scenes/Main.tscn) 的 `SchematicView` 节点加 `clip_contents = true`。
- 或在 [Runtime/ui/schematic_view.gd `_ready`](../../Runtime/ui/schematic_view.gd) 设 `clip_contents = true`。
- 验收：中键拖动把元件移出视口 → 不溢出到左 dock / 底 tab。

### P3 — hit-test 用实际绘制 Rect（S）

- `_draw_placements` 每个 pl 记录 `_draw_rect_cache[uid] = draw_rect_r`。
- `_find_placement_at(px)` 改为遍历 `_draw_rect_cache`，`r.has_point(px)` 命中。
- 清空时机：每次 `_draw_placements` 开头清空（避免 stale）。
- 验收：P1 修完后，点任一元件中心 → 选中。

### P4 — 滚轮 zoom 走 set_zoom（S）

- [Runtime/ui/schematic_view.gd:157-162](../../Runtime/ui/schematic_view.gd#L157) 滚轮分支：
  - `_zoom = clamp(_zoom * 1.15, 0.1, 20.0); queue_redraw()` → `set_zoom(_zoom * 1.15)`
  - 向下同理。
- 验收：滚轮缩放 → StatusBar zoom% 即时变化。

### P5 — PropertiesPanel 旋转 + 回车焦点（S+）

- 诊断 1：手动加 `print_debug` 到 `_on_rot_selected`，确认 OptionButton.item_selected 是否触发。
  - 若触发但 rotation 未变：`_uid` 可能被 reload 清空前 `on_selection_changed` 传入的 `data` 已 stale。
    检查 main_window `_on_schematic_changed` → `view.get_selected_placement()` 返回 post-rotate dict 是否正确。
  - 若未触发：`OptionButton.item_selected` 在 Godot 4.6 是否需要 `selected`（单参）vs `item_selected`（int）。
- 诊断 2：reference 回车后焦点跳到 OptionButton → Enter 冒泡触发下拉。
  - 修复：`_on_reference_submitted` 末尾 `_ref_edit.release_focus()`，或设 `OptionButton.focus_mode = FOCUS_CLICK` 阻止键盘聚焦。
- 验收：手验第 5、6 条通过。

### P6 — CliConsole bbcode 转义（S）

- [Runtime/ui/cli_console.gd:95](../../Runtime/ui/cli_console.gd#L95) `_render_entry`：
  - JSON.stringify(resp) 结果含 `[` / `]` 会被 RichTextLabel 解析成 bbcode tag。
  - 修：`text.replace("[", "[lb]").replace("]", "[rb]")` 先转义，再塞进 `append_text`。
  - 同理 req_text。
- 验收：错误响应红色可见，正确响应绿色可见。

### P7 — 删除 / 连线 undo 补齐（M）

补全 P13 占坑 3 个命令的 inverse：

- **remove_placement** inverse = `[place_component(原 component_ref / reference / pos_nm / rot / mirror), connect(net1, pins1), connect(net2, pins2), ...]`
  - 执行前先读 schematic 把 placement 完整 dict + 涉及其引脚的 net 全部 snapshot，作 inverse。
  - 挑战：re-place 会分配**新** uid（`next_placement_uid` 基于 max），原 uid 无法保留。
    - 缓解：inverse 第一条用 place_component 创建新 uid，再用 set_property 把 reference 改回原值；
      nets 的 pins 引用用 `reference.number` 形式（reference 一致即可还原），无需 uid。
- **connect** inverse = 若结果新建 net（`r.data.pin_count == pins.size()`）→ `remove_net(new_net_id)`；
  若 merge（新增 pins 数 < params.pins.size()）→ `disconnect_pin` for each added pin。
  - 需要 connect 的 `Result.data` 带回 `net_id` 与 `is_new` 标志（commands.gd 补字段）。
- **disconnect_pin** inverse = `connect(affected_net_name, [pin])`
  - 需 disconnect_pin 的 `Result.data` 带回 affected_net_name（现在只带 id）。

改动点：

- [Runtime/modules/schematic/commands.gd `_connect`](../../Runtime/modules/schematic/commands.gd)：Result.data 加 `{net_id, is_new}`
- 同文件 `_disconnect_pin`：Result.data 加 `affected_net_names`
- [Runtime/ui/schematic_view.gd `_delete_selected` / `_apply_wire`](../../Runtime/ui/schematic_view.gd)：push inverse
- 单元：`tests/unit/schematic_edit_primitives_test.gd` 加 "connect result.is_new 标记" + "disconnect Result 含 net_names"
- 集成：`tests/integration/gui_edit_roundtrip_test.gd` 加 remove → inverse 回来 → 状态一致 roundtrip

## 7. 风险与缓解

| 风险 | 概率 | 影响 | 缓解 |
| --- | --- | --- | --- |
| 自动 bbox 对已有非零 bbox 符号有副作用 | 低 | 符号显示错位 | 仅 all-zero 时生效；保留 explicit bbox 优先 |
| OptionButton.item_selected API 差异 | 中 | 旋转仍不工作 | P5 诊断阶段先确认信号签名再改 |
| remove_placement inverse 重建与原 uid 不一致 | 中 | 引用到旧 uid 的旁路代码 break | 所有引用走 reference（非 uid），undo 栈仅用新 uid |
| connect is_new 标记漏情况 | 低 | undo 错误删已有 net | 单元覆盖新建 vs 合并两路径 |
