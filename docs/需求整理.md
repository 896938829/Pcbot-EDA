# Pcbot EDA PCB Maker 需求整理

## 一、核心定位
- **AI作为设计主体**：软件是AI的设计工具，AI驱动完整设计流程
- **人类角色**：审核者、最终用户，通过编辑器界面查看和审核AI设计结果

## 二、AI设计流程支持
### 完整设计环节
1. 元器件文件设计（创建/编辑元件符号和封装）
2. 原理图设计（电气连接设计）
3. PCB布局设计（物理布局规划）
4. Layout执行（布线实现）
5. Review检查（设计规则和电气检查）
6. Debug调试（问题诊断和修复）

### 实时交互要求
- AI通过CLI与编辑器实时交互
- 编辑器提供状态反馈和进度报告机制
- 违反设计规则时在CLI中给出警告

## 三、文件系统设计
### 中间文件需求
- 专门为AI设计的中间文件格式
- 包含原理图和PCB设计的AI操作记录
- 用于AI读取历史决策和更新设计状态
- 支持git版本控制

### 学习机制
- AI可通过读取以下文件学习设计模式：
  - 原理图源文件
  - PCB源文件
  - AI中间文件
  - 编辑器报错日志文件

## 四、接口与集成
### AI访问方式
- **CLI接口**：主要AI访问通道
- **编辑器不调用AI**：仅作为AI的工具，不内置AI功能
- **Skills引导**：编辑器提供操作指南，引导AI Agent正确使用功能

### AI控制环境
- 由用户自行选择的AI程序控制（如Claude Code、Codex等）
- 编辑器完全被动响应AI指令

## 五、性能与规模管理
- 设计规模理论上无限制
- 但需规划性能管理机制，避免大规模设计时性能下降
- 功能模块化，支持按需加载

## 六、开源与协作
- 项目完全开源
- 支持社区贡献和扩展

## 七、人类协作流程
### 审核机制
1. AI完成设计修改
2. 人类通过编辑器UI查看设计文件
3. 人工审核AI设计结果
4. 必要时通过UI手动调整

### 操作支持
- 人类可通过UI操作所有编辑器功能
- 与AI使用相同的功能底层

## 八、状态管理
### 版本控制
- 设计回滚使用git管理
- 编辑器仅提供本次运行时的撤销/重做
- 不管理历史版本，依赖git版本库

### 决策透明度
- AI决策过程由用户的Agent工具自行呈现
- 编辑器只输出操作日志
- 不解释AI的决策逻辑

## 九、设计约束管理
- AI可获取完整的设计规则约束
- 需要元件级别的详细信息（电气参数、物理特性等）
- 规则违反时在CLI中明确警告

---

**总结**：您需要的是一个以AI为设计主体、人类为审核者的EDA工具，通过精心设计的CLI和中间文件格式，让AI能够自主完成从元器件创建到PCB布线的完整设计流程，同时保持人类可读的设计文件和完整的版本控制能力。
