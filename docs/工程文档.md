## 1. 一句话项目说明

**Pcbot EDA** 是一款以 **AI 为设计主体、人类为审核者**的 PCB 设计工具，基于 **Godot Engine 4.6** 构建。AI 通过命令行（CLI）驱动完整的电子设计流程，人类通过编辑器 UI 查看并审核 AI 的设计结果。

## 2. 设计哲学

1. **AI 设计，人类审核**：所有设计操作必须通过 CLI 可达；编辑器不内置 AI 推理能力，仅被动响应 AI 指令。
2. **源文件即 SSOT**：`*.sch.json` / `*.pcbproj` 等稳定序列化的源文件本身就是设计状态真相；历史走 git，不维护重复的快照文件。中间文件（`.pcbot/`）只写别处没有的信息——上次运行的错误、累计未解规则违例。
3. **Git 友好**：所有设计文件采用稳定序列化（键字典序、2 空格缩进、LF 行尾），保证 `git diff` 清晰可读。
4. **精度第一**：内部坐标使用 `int64` 以纳米（nm）为单位，杜绝浮点累积误差。
5. **被动编辑器**：GUI 只呈现与审核，不替 AI 做决策。决策透明度由用户自选的 AI Agent（Claude Code / Codex / 其他）负责呈现。

## 3. 角色与交互模型

```
 ┌───────────────┐        CLI (JSON-RPC 2.0)        ┌──────────────────────────┐
 │  AI Agent     │ ─────── stdin / stdout ───────▶ │  Pcbot (headless Godot)  │
 │ Claude/Codex  │ ◀── stderr（诊断/警告）────────  │  cli/main.gd 入口        │
 └───────┬───────┘                                   └─────────┬────────────────┘
         │                                                     │
         │ 读/写                                               │ 读/写
         ▼                                                     ▼
 ┌─────────────────────────────────────────────────────────────────────────────┐
 │  设计工程目录（git 仓库）                                                     │
 │    源文件：  *.pcbproj  *.sch.json  *.sym.json  *.fp.json  *.sym.svg          │
 │    中间态：  .pcbot/last-run.json  .pcbot/diagnostics.jsonl                   │
 └───────────────────────────────────────────────────────────────────────────── ┘
                                       ▲
                                       │  只读 / 审核 / 手动微调
                                       │
                                ┌──────┴──────┐
                                │  人类用户    │
                                │ Pcbot GUI   │
                                └─────────────┘
```

- **AI Agent** 由用户自由选择（Claude Code、Codex、其他），不属本仓库。
- **Pcbot** 以 **headless** 方式被 AI 短暂调起，执行一条命令后退出；状态完全由中间文件维持。
- **人类**通过同一个 Pcbot 的 GUI 模式打开设计工程进行审核。GUI 与 CLI 共享底层模块，行为一致。

## 4. 顶层目录结构

| 目录 | 职责 |
| --- | --- |
| `Assets/` | 美术资源、3D 模型、贴图、导入配置。大文件走 Git LFS。 |
| `Runtime/` | 运行时 GDScript 代码。按分层组织：`core/`、`modules/`、`io/`、`ui/`。 |
| `Scenes/` | Godot 场景（`.tscn`），多为编辑器 UI 容器与视口。 |
| `cli/`（M1 新增） | CLI 入口与命令注册表，`cli/main.gd` 为 headless 启动点。 |
| `tests/`（M1 新增） | GUT 单元测试与集成测试，按 `Runtime/` 镜像组织。 |
| `addons/`（M1 新增） | Godot 插件，包含 `gut/`。 |
| `docs/` | 工程文档。本文件为入口。 |
| `.godot/` | 引擎缓存，不入 git（`.gitignore` 已配置）。 |

> 当前仓库 `Assets/`、`Runtime/`、`Scenes/` 是空骨架；`cli/`、`tests/`、`addons/` 将在 M1 脚手架任务创建。

## 6. 快速开始

### 6.1 人类用户（GUI）

```bash
# 用 Godot 4.6 打开项目
godot --path .
# 或直接双击 project.godot
```

按 `F5` 运行主场景（M1 暂无主场景；需先跑完 P7 任务）。

### 6.2 AI Agent（CLI / headless）

```bash
# M1 交付后生效
godot --headless --path . -s cli/main.gd -- \
  '{"jsonrpc":"2.0","id":1,"method":"project.new","params":{"path":"./demo"}}'
```

- 单次调用 = 一条 JSON-RPC 命令；stdout 返回 JSON-RPC 响应；stderr 输出日志。
- 退出码约定：`0` 成功 / `1` 用户错 / `2` 系统错 / `3` 违反设计规则。

### 6.3 运行测试

```bash
# Windows / Unix 通用
godot --headless --path . -s addons/gut/gut_cmdln.gd
```

## 9. 词汇表（本仓库高频术语）

| 术语 | 含义 |
| --- | --- |
| **SSOT** | Single Source of Truth。本项目源文件（`*.sch.json` 等）即状态 SSOT；历史 SSOT 为 git。 |
| **中间文件** | `.pcbot/` 下服务 AI 调试与决策的辅助产物：`last-run.json`、`diagnostics.jsonl`、`commit-msg`。**不包含**源文件或快照。 |
| **nm 坐标** | 内部坐标单位纳米，`int64`，避免浮点误差。 |
| **Skills** | YAML 格式的命令使用说明，供 AI 通过 `skills.*` 命令检索。 |
| **headless** | Godot 无窗口模式，CLI 的默认运行方式。 |
| **DRC / ERC** | Design Rule Check / Electrical Rule Check，M2/M3 交付。 |
| **last-run.json** | 每次 CLI 调用后覆盖写，记录 command/exit_code/errors/warnings/touched_files；不入 git。 |
| **diagnostics.jsonl** | 累计规则违例与告警；新记录用 `resolved_by_commit` 标记被消除；入 git。 |
