| 字段 | 值 |
| --- | --- |
| 定位 | 本仓库代码写作的最小共同约定 |
| 读者 | 所有贡献者（AI Agent 与人类） |
| 状态 | draft |
| 最后更新 | 2026-04-16 |

> **M1 原则**：只写"必须一致"的部分。覆盖率门禁、强制 lint、PR 模板等留到 M2+ 再加。

---

## 1. 语言与目录

- **主语言**：GDScript（Godot 4.6）。
- **性能热点**：C++ GDExtension 或 C#（M2+ 才使用；M1 不写）。
- **禁止**：Python / Node.js / 其他脚本作为运行时代码（可作构建时工具，但不入产品运行路径）。

目录职责见 [架构设计 §2](../architecture/架构设计.md#2-系统分层)。

## 2. GDScript 风格

| 项 | 规则 |
| --- | --- |
| 缩进 | **Tab**（Godot 官方风格） |
| 行长 | 建议 ≤ 100 列，硬上限 120 列 |
| 行尾 | LF |
| 文件末尾 | 保留一个空行 |
| 字符串 | 双引号为主；内含双引号时用单引号避免转义 |
| 空行 | 顶层声明之间 2 空行；方法间 1 空行 |

**命名**：

| 实体 | 约定 | 示例 |
| --- | --- | --- |
| 成员 / 函数 / 变量 | `snake_case` | `add_component` |
| 常量 | `UPPER_SNAKE_CASE` | `NM_PER_MM` |
| 类（`class_name`） | `PascalCase` | `ComponentSymbol` |
| 枚举类型 | `PascalCase` | `ExitCode` |
| 枚举值 | `UPPER_SNAKE_CASE` | `USER_ERROR` |
| 信号 | `snake_case`，过去式 / 名词短语 | `component_added` |
| 私有成员（约定） | `_` 前缀 | `_cache` |
| 脚本文件 | `snake_case.gd` | `component_symbol.gd` |
| 场景文件 | `PascalCase.tscn` | `MainWindow.tscn` |
| 节点名 | `PascalCase` | `SchematicView` |
| Autoload 单例 | `PascalCase` | `EventBus` |

**类型标注**：

- 公共 API（对外暴露的函数、`@export` 变量）**必须**写类型。
- 内部临时变量可省略，但若类型不直观（来自字典、网络结果），建议标注。
- 返回 `void` 的函数写 `-> void`。
- 返回可能失败的结果统一用 `Result`（见 §6）。

**`class_name`**：

- 只对**可复用**的 `Resource` / 服务类使用 `class_name`，让它出现在编辑器下拉中。
- 节点脚本（只挂给一个场景）不用 `class_name`，保持场景内聚。

## 3. 模块边界

- `Runtime/modules/<domain>/` 下的脚本只能依赖：
  - `Runtime/core/`
  - 同 `<domain>` 内其他脚本
  - 同模块目录下约定的 `public.gd` 聚合（若有）
- **禁止**跨 `modules/` 直接 `preload` / `load` 对方内部脚本。
- 跨模块通信走 `Runtime/core/event_bus.gd` 或显式的服务注入。
- I/O 层（`Runtime/io/`）**不得**引用领域模型；由上层调用 I/O 并负责映射。

## 4. CLI 命令实现

- 每个命名空间一个目录：`Runtime/modules/<domain>/commands.gd`。
- 注册：
  ```gdscript
  func register(registry: CommandRegistry) -> void:
      registry.add("schematic.place_component", _place_component)
  ```
- 处理函数签名：`func(params: Dictionary) -> Result`。
- 不在处理函数内写 JSON-RPC 协议细节；那是 `cli/jsonrpc.gd` 的事。
- 错误用 `Result.err(code, message, data?)` 返回；不要抛异常穿越边界（Godot 对异常不友好）。

## 5. 坐标与单位

- 内部传递 / 存储 / 比较：`int64` nm。
- **禁止**在跨模块 API 暴露 `float` 坐标；展示层（UI、导出）转换为 mm / mil。
- 常量：
  ```gdscript
  const NM_PER_MM: int = 1_000_000
  const NM_PER_MIL: int = 25_400
  ```
- 辅助函数集中在 `Runtime/core/unit_system.gd`，禁止在业务代码重写换算。

## 6. 错误处理

- **CLI 边界**：统一走 `Result`，最终转成 JSON-RPC 错误对象。
- **内部代码**：
  - 预期失败（用户输入错、文件不存在）：返回 `Result.err` 或显式 bool。
  - 编程错误（不变量被破坏）：`assert()` 或 `push_error()` + 提前返回。
  - **禁止**用空 `pass` 吞错；至少 `push_warning`。
- 错误码枚举在 `Runtime/core/error.gd`；新增需评审避免重复。

## 7. 注释与日志

- **不写解释"做了什么"的注释**——好的命名已经说清楚。
- **写"为什么"的注释**：非显然的约束、绕过的 bug、性能取舍。一行足矣。
- 公共函数用中文 docstring：
  ```gdscript
  ## 在指定页面放置一个元件实例。
  ## 返回放置后分配的 uid；失败时返回 -1。
  func place_component(page_id: String, component_ref: String, pos_nm: Vector2i) -> int:
  ```
- **日志**：
  - 全部经 `Runtime/core/logger.gd`；不要散落 `print()`。
  - 级别：`debug` / `info` / `warn` / `error`；默认过滤为 `info+`。
  - 格式：`<LEVEL> <ISO8601> <module>: <message> <key=value>...`。

## 8. 文件写出规则

### 8.1 源文件（SSOT，入 git）

| 格式 | 规则 |
| --- | --- |
| JSON | 键按字典序；2 空格缩进；UTF-8；末尾 LF 换行；数值不保留无意义 `.0` |
| SVG | `viewBox` 必填；剔除编辑器私有属性；单位 mm |
| SQLite（元件库索引） | **不入 git**；可从 JSON 源重建；表 schema 集中在一处 |

### 8.2 中间文件（`.pcbot/`，服务 AI）

**铁律**：只写"别处没有"的信息。新增任何中间文件须走 ADR 评审。

| 文件 | 规则 | 入 git |
| --- | --- | --- |
| `.pcbot/last-run.json` | 每次 CLI 调用**覆盖写**；字段固定（`ts/command/params/exit_code/errors/warnings/touched_files`）；JSON 键字典序 | 否 |
| `.pcbot/diagnostics.jsonl` | **追加写**；每行严格 JSON；按 `{rule_id, ref}` 去重，仅变化时新增；消除时追加一条带 `resolved_by_commit` 的记录，不修改旧行 | 是 |
| `.pcbot/commit-msg` | 每次成功变更覆盖写；纯文本；AI 可 `git commit -F .pcbot/commit-msg` 直接提交 | 否 |

### 8.3 被禁止的文件类型

- `*.ops.jsonl` / `*.snap.yaml` / `*.snap.json` — 与 git / 源文件重复，不准新增。
- `.tres` / `.res` 作为设计数据 — 仅限引擎场景。
- 任意形如"设计状态投影"、"操作重放缓存"的文件 — 需 ADR 证明"git + 源文件 + `.pcbot/` 无法替代"方可。

违反上述的写出路径视作 bug。

## 9. 测试要求（M1）

- 每个核心 `Resource` 至少 1 个构造用例 + 1 个 round-trip 用例。
- 每个 CLI 命令 1 个 smoke（最小合法参数返回成功）。
- 每个错误码至少被 1 个测试触发。
- **不强制覆盖率数字**；P9 端到端用例必须绿。
- 测试文件：`tests/<mirror_of_Runtime>/<name>_test.gd`，命名 `*_test.gd`。
- GUT 命令：
  ```bash
  godot --headless --path . -s addons/gut/gut_cmdln.gd
  ```

## 10. Git 与提交

### 10.1 分支

- `main`：可发布分支；保持绿。
- `feat/<短名>`：新功能。
- `fix/<短名>`：缺陷修复。
- `docs/<短名>`：纯文档。
- `chore/<短名>`：脚手架、构建、重命名等杂项。
- `refactor/<短名>`：不改行为的重构。

### 10.2 提交信息（Conventional Commits 中文主体）

格式：

```
<type>(<scope>): <中文主体，不超过 50 字>

<可选正文：解释"为什么"，不解释"做了什么">
```

- `type` 取：`feat` / `fix` / `docs` / `chore` / `refactor` / `test` / `perf`。
- `scope` 对应模块或目录：`schematic` / `cli` / `library` / `ui` / `io` / `core` / `docs` 等。
- 一个提交只干一件事。
- 正文给出动机、限制、引用的 issue/ADR。

示例：

```
feat(schematic): 增加网络合并命令

两个同名网络在合并前会校验各自的电气属性是否一致，
不一致时返回 USER_ERROR 而非静默合并，避免掩盖短路。
```

### 10.3 Git LFS

- 3D 模型（`.glb`/`.fbx`/`.obj`）、贴图位图（≥ 1MB）走 LFS。
- 符号 SVG **不走** LFS（保持 git diff 可读）。
- 首次引入 LFS 文件时更新 `.gitattributes`。

### 10.4 不要提交的内容

- `.godot/` 下任何缓存（已在 `.gitignore`）。
- `.pcbotlib.sqlite`（从 JSON 源可重建）。
- `.pcbot/last-run.json`、`.pcbot/commit-msg`（瞬态，每次 CLI 调用被覆盖）。
- 本地机器绝对路径、私人 API Key、`.env`。
- 二进制 `.tres` / `.res` 作为设计数据（可用于引擎场景配置，不用于设计工程）。

### 10.5 .gitignore 对 `.pcbot/` 的处理

仓库 `.gitignore` 需要：

```
.pcbot/last-run.json
.pcbot/commit-msg
!.pcbot/diagnostics.jsonl
```

`diagnostics.jsonl` 是项目质量史，必须入 git。

## 11. 禁止项速览

- `OS.execute` 调用任意外部二进制（除非在构建脚本中）。
- 在热路径用 `float` 坐标做几何计算。
- 跨模块 `preload` 他人内部脚本。
- 把引擎警告静默化：`@warning_ignore` 必须附一行"为什么"注释。
- 在运行时代码嵌入 AI 推理（违反"编辑器被动响应"原则）。

## 12. 可选工具（推荐但不做门禁）

- [`gdformat`](https://github.com/Scony/godot-gdscript-toolkit) — 自动格式化。
- [`gdlint`](https://github.com/Scony/godot-gdscript-toolkit) — 静态检查。
- 编辑器设置：VSCode `godot-tools` 扩展 或 Godot 内置脚本编辑器。

M2 起可能把 `gdformat --check` 加到 CI；现阶段靠 review。

## 13. 版本兼容策略

- 文件格式顶层加 `"format_version": <int>`（见架构设计）。
- 破坏性变更同时：
  1. 提交迁移器 `Runtime/io/migrations/v<N>_to_v<N+1>.gd`；
  2. 升级本规范文档与 Skills YAML；
  3. 在 commit 正文明示"BREAKING"。
